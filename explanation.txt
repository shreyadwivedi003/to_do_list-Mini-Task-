To-Do List API Documentation & Architecture Overview
1. Executive Summary
This document provides the technical specifications, architectural reasoning, and implementation challenges for the Node.js/Express-based To-Do List (Notes) API. The system is designed to handle core task management through standard RESTful constraints.
2. CRUD Operations Overview
The API collectively fulfills the four fundamental operations of persistent storage (CRUD) using an in-memory data array (notes):
Create (C): Handled via POST /notes. It appends new task objects containing user-defined data directly into the system array.
Read (R): Handled via GET /notes. It retrieves the entire state of the array, enabling user interfaces to display updated task data.
Update (U): Handled via PATCH /notes/:index. It targets a specific element using its position index, allowing selective data modifications without replacing the whole resource.
Delete (D): Handled via DELETE /notes/:index. It systematically eliminates a single record from the data array and shifts remaining records to maintain array integrity.
3. Detailed API Endpoint Specifications
📥 1. Create a Note
Endpoint URL: http://localhost:3000/notes
HTTP Method: POST
Reasoning: POST is used because it is the standard REST method for submitting data to create a new resource. The server processes the payload and appends it to the collection.
Request Body Format (JSON):

✏️ 3. Update a Note
Endpoint URL: http://localhost:3000/notes/:index (e.g., /notes/0)
HTTP Method: PATCH
Reasoning: Unlike PUT (which expects a complete replacement of the resource), PATCH is chosen for partial updates. If a user only sends a title, the existing description remains safe and untouched.
Request Body Format (JSON):
JSON
{
  "title": "Finish Advanced DSA Practice"
}
Expected Responses:
Success (200 OK):
JSON
{
  "message": "Note updated successfully",
  "note": { 
    "title": "Finish Advanced DSA Practice", 
    "description": "Solve 3 LeetCode problems on arrays and graphs." 
  }
}
Error (404 Not Found): Sent if the target index is out of bounds or negative.
JSON
{ 
  "message": "Note not found" 
}



🗑️ 4. Delete a Note
Endpoint URL: http://localhost:3000/notes/:index (e.g., /notes/0)
HTTP Method: DELETE
Reasoning: Maps precisely to the deletion of a resource. Using a route parameter (:index) cleanly specifies which exact resource must be removed.
Request Body Format: None.
Expected Responses:
Success (200 OK):
JSON
{ "message": "Note deleted successfully" }
